## 1. 项目建设流程

### 1.1 准备阶段

| 步骤 | 操作 | 检查项 |

| 1.3 | 检查 SX1278 模块 | 接线正确, 天线完好 |

## 5. 完整功能测试

按照实际使用场景, 从头到尾走一遍:

```
测试步骤:
  1. 从机上电 → 观察继电器默认关闭
  2. 主机上电 → 屏幕显示 "1# 温控 V3.0"
  3. 等待 5 秒 → 主机显示温度数据
  4. 按 KEY5 → 开启烟感检测, 屏幕显示 "SM:ON"
  5. 模拟烟雾 → 点燃打火机靠近 MQ-2, 屏幕显示 "SM:ALM"
  6. 移开打火机 → 屏幕恢复 "SM:ON"
  7. 按 KEY3 3 次 → 阈值设为 33°C
  8. 用吹风机加热传感器 → 温度升到 33°C 后风扇自动启动
  9. 移开吹风机 → 温度降到 32°C 后风扇自动关闭
  10. 按 KEY2 → 切换到手动模式 (屏幕显示 1#MANU)
  11. 按 KEY1 → 手动开启风扇
  12. 按 KEY1 → 手动关闭风扇
  13. 按 KEY2 → 切回自动模式
  14. 拔掉从机电源 → 10 秒后主机显示 "OFFLINE"
  15. 重新插上从机电源 → 自动恢复
```

### 6.3 中断资源检查

| 中断源 | 优先级 | 使用者 | 冲突检查 |
|--------|--------|--------|----------|
| SysTick | 默认 | SysTick_Handler (1ms 时钟) | ✅ |
| EXTI0 | 抢占=1, 子=0 | LoRa DIO0 → SX1278 IRQ | ✅ |
| TIM2 | 抢占=2, 子=0 | HUB12 扫描刷新 | ✅ |

**结论: 中断资源无冲突 ✅**
- EXTI0 优先级 (1) 高于 TIM2 (2), 确保 LoRa 中断不被显示刷新打断
- SysTick 使用默认优先级, 不影响其他中断

## 7. 常见问题排查

### 7.1 编译错误

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| `hub75.h: No such file` | 旧代码仍引用 hub75 | 确认所有 `#include "hub75.h"` 已改为 `#include "hub12.h"` |
| `MODE_AUTO undeclared` | 缺少宏定义 | 确认 `app_master.h` 中包含 `#define MODE_AUTO` |
| `undefined reference to TIM2_IRQHandler` | 启动文件未包含 | 检查 startup_stm32f10x_md.s 是否在工程中 |
| `undefined reference to EXTI0_IRQHandler` | 同上 | 同上 |
| `mq2.h: No such file` | 缺少 MQ-2 驱动 | 确认 `mq2.h` 和 `mq2.c` 已添加到工程 |

### 7.2 硬件问题

| 现象 | 可能原因 | 排查步骤 |
|------|----------|----------|
| DS18B20 返回 -100°C | 未接 4.7kΩ 上拉 / 虚焊 | ① 万用表测 PA0 对 3.3V 阻值 ≈ 4.7kΩ ② 示波器看复位脉冲 |
| MQ-2 一直报警 | 灵敏度太高 / 环境烟雾 | ① 调节模块电位器 ② 在干净空气中测试 |
| MQ-2 无响应 | DOUT 未正确连接 | ① 检查 PA1 接线 ② 万用表测量电平 |
| LoRa 版本号不是 0x12 | SPI 接线错误 / 模块坏 | ① 逐一检查 PA4~PA7 ② 换一个模块测试 |
| LoRa 无通信 | 天线未接 / 频率不匹配 | ① 必须接天线 ② 两台设备用相同配置 |
| 继电器不动作 | GPIO 驱动不足 | ① 万用表测 PB1 ② 加三极管驱动电路 |
| P10 屏不亮 | 5V 未接 / OE 信号错误 | ① 确认 5V 供电 ② 用示波器检查 CLK 信号 |
| P10 屏闪烁 | TIM2 刷新率不对 | ① 检查 TIM2 预分频和周期 ② 确认中断正常触发 |
| P10 级联只亮一半 | 级联连接错误 | 检查面板1的DOUT是否连接到面板2的DIN |
| KEY1 (PC13) 不灵敏 | 无内部上拉 | 外加 10kΩ 上拉电阻 |
| KEY5 (PB9) 无响应 | PB9 被占用 | 检查 PB9 是否被其他功能使用 |

### 7.3 软件问题

| 现象 | 可能原因 | 排查步骤 |
|------|----------|----------|
| 主循环卡死 | `while(1)` 等待初始化 | 在死循环处加 LED 闪烁, 定位卡在哪个初始化 |
| 显示内容错乱 | 帧缓冲越界 | 检查 `HUB12_SetPixel()` 的边界检查 |
| 按键连发 | 消抖时间不够 | 增加 `KEY_DEBOUNCE_MS` |
| 从机频繁重启 | 电源不稳 | 加大电容 (100μF + 100nF) 在 STM32 电源引脚旁 |
| 浮点运算慢 | 没有 FPU (Cortex-M3) | 可以用整数运算代替 (温度×10 存为整数) |
